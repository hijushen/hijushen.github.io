可以方便的 遍历容器内元素 。  可以简单的理解为指针，游标。指在第一个元素左边。Iterator。
有继承，重写，多态，父类引用指向子类对象。 返回一个Iterator接口。
boolean hasNext(); //判断游标右边是不是有元素；
Object next(); 返回游标右边的元素并将游标移动到下一个位置；
void remove();  删除游标左边的元素，在执行玩next之后，该操作只能执行一次。
Iterator对象的remove方法是在迭代过程中删除元素的唯一方法。

增强的for循环
import java.util.*;

public class EnhancedFor {
     public static void main(String[] args) {
          int[] arr = {1, 2, 3, 4, 5};
          for(int i : arr) {
               System.out.println(i);
          }
         
          Collection c = new ArrayList();
          c.add(new String("aaa"));
          c.add(new String("bbb"));
          c.add(new String("ccc"));
          for(Object o : c) {
               System.out.println(o);
          }
     }
}


缺陷：不能方便的访问 下标值，和Iterator相比，不能方便的删除 元素。 （在内部也是调用Iterator）
除了简单遍历并读出其中内容，没有什么有点，不建议使用。
