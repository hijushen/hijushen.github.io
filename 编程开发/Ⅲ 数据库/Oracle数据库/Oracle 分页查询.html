1. SELECT *
  2.   FROM (SELECT A.*, ROWNUM RN
  3.           FROM (SELECT TO_CHAR(P.RELEASETIME_DT, 'yyyy-MM-dd') 时间, P.*
  4.                   FROM PANA.PFSA_PANA_TRACE_PLACE P
  5.                  ORDER BY P.RELEASETIME_DT ASC) A --第一层,原始SQL
  6.          WHERE ROWNUM <= 1000) --第二层,对原始SQL控上线.
  7.  WHERE RN > 900 --第三层,控下线.

二,  优化的查询方式: Cost 差一个数量级.
  1. SELECT *
  2.   FROM (SELECT TO_CHAR(A.RELEASETIME_DT, 'yyyy-MM-dd') 时间
  3.           FROM (SELECT P.*,
  4.                        ROW_NUMBER() OVER(ORDER BY P.RELEASETIME_DT ASC) AS RN
  5.                   FROM PANA.PFSA_PANA_TRACE_PLACE P) A --第一层,使用ROW_NUMBER() OVER分析函数.
  6.          WHERE A.RN > 5000) QQ --第二层完成起点预设,如从5000行开始
  7.  WHERE ROWNUM <= 30 --第三次完成行数测试
