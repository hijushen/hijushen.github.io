###  根据存储过程名字 显示获得 其内容
`exec sp_helptext 'P0GeneralOrgnGet'`
### 查询表名的相应属性:
`sp_help [A2AccOrg]`  `sp_columns [A2AccOrg]` 都是可以的, 替代了mysql和oracle中的 `exec table1`
### 根据存储过程内容, 查询存储过程名字
```
  1. select s.name,c.text,*
  2. from sysobjects s inner join syscomments c on c.id=s.id 
  3. where c.text like '%a0org%'
```
### 查询数据库中存储过程创建\修改信息(查看变动)
- 主要是`sys.objects`的使用, 基本相同包括`sys.sysobjects`. 两者不包括`sys.triggers`.
- 类似的,包括`sys.all_objects`.范围比上面的要多.
- [MSDN地址](https://msdn.microsoft.com/zh-cn/library/ms190324(v=sql.120).aspx)
```
  1. select a.* from sys.objects as a
  2. where a.type_desc = 'SQL_STORED_PROCEDURE'
  3. order by a.modify_date desc,a.create_date desc
```

SELECT * FROM dbo.V0DataDict WHERE name='表名'

EXEC sp_UsersGetByOrg '机构Id或机构名称'  --根据机构ID或机构名称得到机构下面的所有用户信息 
 PB1GetUpOrgListByOrgID
