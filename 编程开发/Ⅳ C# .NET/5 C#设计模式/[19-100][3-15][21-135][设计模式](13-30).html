代理模式.
抽象主题角色（Person）：声明了真实主题和代理主题的公共接口，这样一来在使用真实主题的任何地方都可以使用代理主题。
代理主题角色（Friend）：代理主题角色内部含有对真实主题的引用，从而可以操作真实主题对象；代理主题角色负责在需要的时候创建真实主题对象；代理角色通常在将客户端调用传递到真实主题之前或之后，都要执行一些其他的操作，而不是单纯地将调用传递给真实主题对象。例如这里的PreBuyProduct和PostBuyProduct方法就是代理主题角色所执行的其他操作。
真实主题角色（RealBuyPerson）：定义了代理角色所代表的真是对象。
 
附：在实际开发过程中，我们在客户端添加服务引用的时候，在客户程序中会添加一些额外的类，在客户端生成的类扮演着代理主题角色，我们客户端也是直接调用这些代理角色来访问远程服务提供的操作。这个是远程代理的一个典型例子。

1--代理类: 
  1.     public abstract class Person
  2.     {
  3.         public abstract void BuyProduct();
  4.     }
  6.     public class RealBuyPerson:Person
  7.     {
  8.         public override void BuyProduct()
  9.         {
  10.             Console.WriteLine("帮我买一个IPhone和苹果电脑");
  11.         }
  12.     }
  14.     public class Friend:Person
  15.     {
  16.         RealBuyPerson realByP;
  17.         public override void BuyProduct()
  18.         {
  19.             Console.WriteLine("通过代理类访问真实的对象");
  21.             
  22.             if (realByP  == null)
  23.             {
  24.                 realByP = new RealBuyPerson();
  25.             }
  27.             //1-代理前做事.
  28.             this.PreBuyProduct();
  30.             realByP.BuyProduct();
  32.             //3-代理后做事.
  33.             this.PostBuyProduct();
  34.         }
  36.         public void PreBuyProduct()
  37.         {
  38.             Console.WriteLine("事前列清单...");
  39.         }
  41.         public void PostBuyProduct()
  42.         {
  43.             Console.WriteLine("事后对清单...");
  44.         }
  45.     }
2--客户端调用.
  1.     class Program
  2.     {
  3.         static void Main(string[] args)
  4.         {
  5.             Person proxy = new Friend(); //代理类Friend就开始了
  7.             proxy.BuyProduct();
  8.             Console.Read();
  9.         }
  10.     }
