观察者模式. < http://www.cnblogs.com/zhili/p/ObserverPattern.html  >
C#中有委托和时间 简化了订阅者接口的调用.
适用条件:

观察者模式有以下几个优点： 
观察者模式实现了表示层和数据逻辑层的分离，并定义了稳定的更新消息传递机制，并抽象了更新接口，使得可以有各种各样不同的表示层，即观察者。
观察者模式在被观察者和观察者之间建立了一个抽象的耦合，被观察者并不知道任何一个具体的观察者，只是保存着抽象观察者的列表，每个具体观察者都符合一个抽象观察者的接口。
观察者模式支持广播通信。被观察者会向所有的注册过的观察者发出通知。
　　观察者也存在以下一些缺点： 
如果一个被观察者有很多直接和间接的观察者时，将所有的观察者都通知到会花费很多时间。
虽然观察者模式可以随时使观察者知道所观察的对象发送了变化，但是观察者模式没有相应的机制使观察者知道所观察的对象是怎样发生变化的。
如果在被观察者之间有循环依赖的话，被观察者会触发它们之间进行循环调用，导致系统崩溃，在使用观察者模式应特别注意这点。

  1.     class Program
  2.     {
  3.         static void Main(string[] args)
  4.         {
  5.             TenXun tenXun = new TenXunGame("TenXun Game","Have a new game published ....");
  7.             //添加订阅者
  8.             tenXun.AddOberserver(new Subsriber("Learning Hard"));
  9.             tenXun.AddOberserver(new Subsriber("Me"));
  11.             tenXun.Update();
  13.             Console.ReadKey();

  18.     public abstract class TenXun
  19.     {
  20.         private List<IObserver> observers = new List<IObserver>();
  22.         public string Symbol { get; set; }
  23.         public string Info { get; set; }
  24.         public TenXun(string symbol,string info)
  25.         {
  26.             this.Symbol = symbol;
  27.             this.Info = info;
  28.         }
  30.         #region 订阅号维护
  31.          public void AddOberserver(IObserver ob)
  32.         {
  33.             observers.Add(ob);
  34.         }
  36.         public void RemoveObserver(IObserver ob)
  37.          {
  38.              observers.Remove(ob);
  39.          }
  40.         #endregion
  42.         public void Update()
  43.         {
  44.             foreach(IObserver ob in observers)
  45.             {
  46.                 if (ob!= null)
  47.                 {
  48.                     ob.ReceiveAndPrint(this);
  49.                 }
  50.             }
  51.         }
  52.     }
  54.     public class TenXunGame:TenXun
  55.     {
  56.         public TenXunGame(string Symbol,string info) :base(Symbol,info)
  57.         {
  59.         }
  60.     }
  62.     public interface IObserver
  63.     {
  64.         void ReceiveAndPrint(TenXun tenxun);
  65.     }
  68.     public class Subsriber:IObserver
  69.     {
  70.         public string Name { get; set; }
  71.         public Subsriber(string name)
  72.         {
  73.             this.Name = name;
  74.         }
  76.         public void ReceiveAndPrint(TenXun tenxun)
  77.         {
  78.             Console.WriteLine("Notified {0} of {1}'s " + " Info is: {2}", Name, tenxun.Symbol, tenxun.Info);
  79.         }
  80.     }
