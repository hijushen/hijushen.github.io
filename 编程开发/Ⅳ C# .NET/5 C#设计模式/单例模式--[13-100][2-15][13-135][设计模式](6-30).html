单例模式:
练习问题, 常错点:
1) 单例private 构造函数.
2)同1)后, 那么创建只能通过类级访问. 那么方法和返回对象就一定是static 的了.
3)饿汉模式中居然可以new.!!!
4)懒汉模式中先为null, 然后双重锁定. 判定为空, 锁定, 再判定. 双重锁定.....
如下为具体; 
1, 饿汉模式:
特点:  ü无须考虑多个线程同时访问的问题；调用速度和反应时间优于懒汉式单例；资源利用效率不及懒汉式单例；系统加载时间可能会比较长
  1.     class EagerSingleton
  2.     {
  3.         private static EagerSingleton instance = new EagerSingleton();
  4.         private EagerSingleton()
  5.         {
  7.         }
  8.         public static EagerSingleton GetInstance()
  9.         {
  10.             return instance;
  11.         }
  12.     }
2, 懒汉模式. 使用双重锁定的方式.
ü懒汉式单例类：实现了延迟加载；必须处理好多个线程同时访问的问题；需通过双重检查锁定等机制进行控制，将导致系统性能受到一定影响
