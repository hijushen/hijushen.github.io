1）当对一个类应用 sealed 修饰符时，此修饰符会阻止其他类从该类继承和abstract相排斥。 在下面的示例中，类 B 从类 A 继承，但是任何类都不能从类 B 继承。实现第三方类库不想被客户端继承，或没有必要的类防止滥用造成层次体系混乱。恰当使用可以提高运行效率，因为可以不用考虑运行效率问题。

    class A {}    
    sealed class B : A {}

还可以在重写基类中的虚方法或虚属性的方法或属性上使用 sealed 修饰符。 这将使您能够允许类从您的类继承，并防止它们重写特定的虚方法或虚属性。

2）Static 修饰的变量为静态变量，在类初始化时分配空间，通过类访问，不使用static修饰的为非静态变量，在对象初始化时分配，每个对象包含的值不同；

3）const和static readonly的区别：都是常量，使用方法相同，static readonly在运行期()初始化，const在编译器初始化；

using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;

namespace @delegate
{
    public class SamplesDelegate
    {

        public delegate String myMethodDelegate(int myInt);

        //声明一个字符类型委托，但其所带参数可为任何类型，本例为证型

        public class mySampleClass
        {

            public String myStringMethod(int myInt)
            {
                if (myInt > 0)
                    return ("positive" );
                if (myInt < 0)
                    return ("negative" );
                return ("zero" );
            }

            public static String mySignMethod( int myInt)
            {
                if (myInt > 0)
                    return ("+" );
                if (myInt < 0)
                    return ("-" );
                return ("" );
            }
        }

        public static void Main()
        {
//核心用法如下：在实例化过程中使用的方法：
            mySampleClass mySC = new mySampleClass();
            myMethodDelegate myD1 = new myMethodDelegate(mySC.myStringMethod);
            myMethodDelegate myD2 = new myMethodDelegate (mySampleClass.mySignMethod);

            Console.WriteLine("{0} is {1}; use the sign \"{2}\"." , 5, myD1(5), myD2(5));
            Console.WriteLine("{0} is {1}; use the sign \"{2}\"." , -3, myD1(-3), myD2(-3));
            Console.WriteLine("{0} is {1}; use the sign \"{2}\"." , 0, myD1(0), myD2(0));
            Console.ReadLine();
        }

    }


    /*
    This code produces the following output:
 
    5 is positive; use the sign "+".
    -3 is negative; use the sign "-".
    0 is zero; use the sign "".
    */
}
