CTS类型系统     公共语言规范CLS
程序集：.Net 部署为程序集（Assembly）的逻辑单元，可以是一个类库(dll)，一个控制台，或Windows应用程序，一个Windows服务，甚至是一个Asp.NET。
可以包含一个或者多个物理文件，完全在内存中的动态程序集，可不包含任何文件。

元数据：实际的代码外，程序所包含内容称为元数据；
     包含：程序集元数据---（程序集，版本，生成号码，程序清单）；
           类型元数据---（程序集中定义的类型，公共成员信息）；
     默认情况下，程序集私有，只能从同一目录运行代码中访问；
     如果把程序集存储在全局程序集缓存（GAC），可以共享程序集；
     共享程序集必须使用一个证书进行签名，保证他们不被其他程序破坏。

.Net和COM无缝的互操作          .Net程序集生成COM类型库；
                               COM类型库，生成Net包装器。

Framework 类库     继承自Windws API 函数功能，提高高级别操作，如数据访问XML串行化，字符串与集合处理；

Windws窗体     System.Windows.Forms命名空间
Web窗体      Windows窗体在ASP.NET中的等效形式
ADO.NET     .NET的数据访问API，简单的对象模型，提供可以连接数据源，执行命令并检索，操作数据，相对应ADO而言，多支持了XML。XML作为存储和传送数据的方法

Remoting     代理/存根系统在进程边界之间进行调用，消息通过一系列接收器在客户/服务器之间传递；每个接收器可以以某种方式（消息串行化为二进制活SOAP格式），Remoting 允许以任何格式和任何传送方式跨进程调用。

.NET FRAMEWORK REDISTRIBUTABLE  ---完成框架本身，类似于JRE
.NET FRAMEWORK SDK(软件开发工具包)---完整框架学习.NET所需实例，指南，约130Mb。
