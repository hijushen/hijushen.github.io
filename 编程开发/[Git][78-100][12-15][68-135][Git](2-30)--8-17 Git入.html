工作区, 缓存区, 版本库的区别.
git add 之后进入到缓存区.
修改之后的内容撤销问题. 
使用命令, 回退工作区内容, 用缓存区或者版本库内替换.
  1. git checkout -- test.txt //更新缓存区或者库存区到工作区,缓存区不空也就是刚刚add的就回退这个. 算就近覆盖吧!!二者情况.... 保证和版本库一样,就跟如下一样, 第一步把缓存区清了,第二次更新版本库到本地.
如上命令文件回到最后一次git commit 或者git add之后的状态.

二)如果已经提交到缓存去, 完全回退到版本库的过程需要2步走.
将缓存区内容清空, HEAD^ 等上一个版本. HEAD为库中当前版本. 
作用等于  清空当前缓存去内容:
  1. 1.git reset HEAD test.txt 将缓存区置换到工作去
  2. 2.git checkout -- text.txt //将版本区更新到工作区
然后使用第一步, 替换工作区内容, 目前缓存区为空, 所以版本区替换.

文件的删除动作:
本地文件删除: rm test.txt. 对版本库没有影响. 
影响到暂存库的操作:
  1. git rm test.txt
此时可以回退, 就是清空暂存区而已, 变成本地没有, 版本库里面依然存在的情况. 提示删除.
继续删除动作就是 git commit -m "del test.txt"即可．
如果不想删除，就是将缓存区清空．此时变成仅仅删除本地操作, 本地文件未恢复. 命令: git reset HEAD test.txt回退到当前版本.
然后git checkout test.txt 这样文件就回来了.
