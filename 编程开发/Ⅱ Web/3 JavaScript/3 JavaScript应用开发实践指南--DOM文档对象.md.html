## 文档对象模型
1. DOM结构树, 分三种类型的节点: **元素** 节点如`<html></html><head></head>`等, ** 文本节点** 如`<p>我是文本节点</p>`中的内容, **属性节点** 如:`<div id="header"></div>`中`id="header"` 就是属性节点, **文本节点** 和**属性节点** 都没有子元素. 
## 处理元素节点
如何访问元素节点,只要是DOM里面的节点都可以抓取出去,用JavaScript对其进行存储或者修改. **降低定位到文档内容所需要的节点数量, 学会机种不同DOM访问的方式, 找出最有效率的 一点** 创建优秀的**用户体验**, 对节点的访问**越快也好**.
- 根据ID定位元素. `document.getElementById('header');` 等,最有效率的方式.
- 根据标签名定义多个,**一组元素**.:`getElementsByTagName`
- 根据class属性定位元素. Js中的支持与Css的使用相比要相对较差. `document.getElementsByClassName('dropcap huge');`. 同上面一样返回`Elements`的方法都是返回Nodelist类型的元素,通过item(0)或者[0]数组下标来完成类别组内容的获取.
- 在JavaScript中使用CSS 选择器来定位节点. `querySelectorAll()`和`querySelector()`方法,前者返回NodeList之类的数组,而后者只返回符合条件的第一个元素.
```
  1. //get the header ID element
  2. document.querySelector("#header");
  3. //get the first element with a dropcap class
  4. document.querySeletor(".dropcap");
  5. //get all the paragraphs with a "dropcap" class - produces a nodelist
  6. document.querySelectorAll(".dropcap");
  7. //get all elements with a class of "dropcap" or "huge"
  8. document.querySelectorAll(".dropcap, .huge");
  9. //get all paragraphs that have a class
  10. document.querySelectorAll("p[class]");
```
## 处理属性节点
- 采用方法`getAttribute()`,`setAttribute`,`removeAttribute`,`hasAttribute`.
最佳实践规则,**属性节点存在是再操作**, **不要对一个不可能执行的Js代码浪费时间**
```
  1. if(document.getElementById("plc").hasAttribute("class")){
  2.   document.getElementById("plc").getAttribute("class");
  3.   //set Attribute方式--replace
  4.   document.getElementById("plc").setAttribute("class","hidden");
  5.   //set Attribute方式--add a new attribute
  6.   document.getElementById("plc").setAttribute("title","we move this element off screen");
  7. }

```
## 处理文本节点
获取元素节点后使用`innerHTML`方式获取. 即 `document.getElementById("target-area").innerHTML = "<p>hello world</p>";`

## 遍历DOM
- `parentNode, previousSibling nextSibling fistChild lastChild` 来完成元素节点的访问, 
## 增加或者删除节点
- `createElement(), createTextNode(), appendChild(), removeChild()` 向Dom中增加元素时,前三种方法共用.
