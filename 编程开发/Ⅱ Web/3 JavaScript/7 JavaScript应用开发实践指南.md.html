## 代码组织方式
### 通用代码风格规范
- 作用域. 清理全局作用域..
- 遇到错误尽快终止.
### 代码设计
- 文档和目录结构. js下自写代码和引入代码的规范. CSS文件同样可以这么组织.
    - js
        - application.js
        - lib
            - libary.js
            - library.plugin.js

- 文档内脚本, 可以采用**自动化**的源文件拼接技术.
- 变量声明
    - 所有变量放在其函数的头部.
    - 添加清晰而明确的注释
    - 使用协调一致的变量分组模式(一个,号包含了几个变量的声明)
    - 变量与函数的命名
    - 语句内分割. {} 前后应该有空格
    - 断行(超过80字符一行会增加理解的负担)
    - 操作符前后留空格, 如: `a === b`等.
    - 不要用`eval()`, 会遭受攻击? 使用`json.parse()`等方法.

### 代码结构
- 函数
- 匿名函数
- 变量形式出现的函数:`var pizza = function(){};`, 功能和函数相同, 2种不同的风格而已.
- 方法形式出现的函数: `var party = {pizza:function(){},be:function(){}};`,无区别,需要指定对象后使用方法.

### JavaScript开发模式
- 函数与闭包模型
- 事件驱动模式
- 二者最大的区别是事件驱动模式只是提供函数去执行, 而闭包采用提供api的形式去调用.
### 习题
- 构建JavaScript程序时所要完成的4件事情
    - 写出能够运行的代码; 写出能够良好运行的代码;让代码变得易读;尽量提升代码的可复用性.
- 函数或者变量名不宜使用下划线开头
    - 某些编程语言中, 下划线开头表示私有部件, 而JavaScript中没有这个概念.
- 匿名函数的最佳使用时机是什么
    - 某些代码打算一次使用时.
