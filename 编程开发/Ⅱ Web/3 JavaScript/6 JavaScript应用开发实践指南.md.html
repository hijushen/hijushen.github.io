以事件处理实现用户交互. 处理事件模型:
1. 获取DOM节点;
2. 在其中设置事件处理程序;
3. 如果事件被触发, 那么就执行用于处理该事件的特定函数.

> **事件驱动式**JavaScript编程(event-driven JavaScript). 还包括: MVC, 面向对象模型.

## 设置事件处理程序
- 事件处理器: `btn.onclick = function(){}` , 包括`onsubmit`,`onchange`,`onfocus`,`onblur`,`onmouseover`等事件处理程序.
    - 优点是简单, 但是无法复制两种事件处理程序, 具有局限性. 推荐**事件监听器**替代之.
- 事件监听器: 
```
  1. var btn = document.getElementById('btn');
  3. if(btn.addEventListener){ //IE8之前的版本检测
  4. 	btn.addEventListener("click",function(){
  5. 		alert('clicked the button');
  6. 	},false) //false:是否启用捕获模式,99%的情况为false
  7. } else {
  8. 	btn.attachEvent('click',function(){
  9. 		alert('Clicked the button');
  10. 	});
  11. }
    //移除事件监听器
  1. if(btn.removeEventListener){
  2. 	btn.removeEventListener("click",alertMessage,false);
  3. } else {
  4. 	btn.detachEvent("click",alertMessage);
  5. }

```

## 鼠标与键盘事件
- `click`事件. 只有单击时才执行,没有`focus`这样,只要集中了就触发事件.
- `focus`事件:输入框或者Tab键切换到, 如果离开处于非激活正太,则是`blur`事件.
- `change`事件.
- `mouseover` `mouseout` 鼠标悬停事件.
- 阻止浏览器默认行为. `event.preventDefault()`
- `keydown` `keypress` `keyup`事件

## 触摸及方向改变事件
- `touchstat` `touchend`事件.
- `touchmove`事件. 扫屏手势(swipe gesture).或者在屏幕上对各个物件的拖拽功能.
- `orientationchange`事件, 屏幕方向发生了改变触发的事件.在支持加速计的设备上发生.

##习题问答
- 事件处理器与事件监听器的区别: 处理器只能增加一个, 而监听器没有这个限制.
- IE8及早期浏览器版本中的事件监听器增加方式: `attachEvent()`
- preventDefault()方法用途: 组织默认行为,譬如link `href`的跳转, `submit`而不提交数据.
