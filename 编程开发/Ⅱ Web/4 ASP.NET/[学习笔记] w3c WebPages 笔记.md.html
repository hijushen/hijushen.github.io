## Web Pages    
### 简介
- 三种ASP.NET 一种: 其他2种: mvc,web Form.

### WebPages Razor
- 代码块:`@{}`
- 内联表达式(函数和变量)以@开头,代码语句用分号结束
- 变量用`var`关键字声明, 字符串用引号括起来, C#代码区分大小写, C#文件的拓展名是`.cshtml`

### ASP.NET Web Pages页面布局
- 一致的外观, 相同头部, 底部, 样式和布局.
- 内容块: `Content Blocks`. `@RenderPage()` 用于引入其他页面文件
- 布局页: `Layout Page`, `RenderBody()`用于模板页中预留其他页面位置, 其他页面引入模板页之后, 编写的内容就显示在模板对应位置中. 引入的语法采用的是: `@{Layout="Layout.cshtml"}`
- 防止文件被浏览: 采用`_Layout.cshtml``_header.cshtml``_footer.cshtml`, 以`_`开头的文件在ASP.NET中禁止被文件浏览.
- 隐藏敏感信息: 数据库密码, 电子邮件密码等最通用的办法保存在`_AppStart`的单独的文件中.内容: `@{WebMail.SmtpServer="mailserver.example.com";....}`等.

### WebPages文件夹
- `~`运算符 代表虚拟路径. 代码中使用的路径
- `Server.MapPath()` 将虚拟路径映射为 -> 服务器能理解的物理路径, 用于打开数据库上的文件时就这么做.
- `Href`方法: 将代码中的路径 -> 浏览器可以理解的路径

### WebPages全局
- 在Web启动之前: `_AppStart`.
- 每个页面之前:`_PageStart`. 每个文件中任何页面之前运行的代码.

### WebPages表单
- 用到相应代码: `Request["Choise"]`, 读取Choice的值

### WebPages对象
- `Page`对象: `RenderPage()`,`RenderBody()`. 对象属性: `isPost`, `Request[""]`.
- 某些Page对象方法: `href`:使用指定的值创建URL. `Write(object)` 将对象作为HTML编码字符串写入, `WriteLiteral` 写入对象时优先使用HTML编码.
- Page属性: 可以在内容文件中设置, 然后再布局文件中使用:　`@{Page.Title = "Home Page"}`, 布局文件中使用方式: `@Page.Title`.

### WebPages文件
- 读取文件显示中: `@dataItem <text>&nbsp;</text>` 使用空的占位符.

### WebPages 帮助器
- `WebGrid` 帮助器等; Chart帮助器, WEbMail帮助器, WebImage帮助器
- `var grid = new WebGrid(data); @grid.GetHtml()`的方式获得表单
