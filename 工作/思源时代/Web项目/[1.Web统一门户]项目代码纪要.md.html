## 常用方法封装成拓展方法篇
- 封装常用方式:
- ![webCommon 下内容](http://i.imgur.com/GP7iKZ8.jpg)
- 序列化与反序列化成对应目标实体:
    - 使用代码: `dt.Serialize().Deserialize<List<A12SSO_Org>>();`. 其中dt为`DataTable`类型数据, 其他数据也可以.
    - 序列化内容:  核心就是采用JSON 序列化的方式: `JSON.Serialize(obj);`, 通过对异常和null的健壮性guard编程;
    - 反序列化内容: 核心采用反射创建对象,然后`JSON.Deserialize<T>(str);`方式生成对应类型. 其中<T>的约束:`public static T Deserialize<T>(this string str) where T : new()`;  尝试进行的反射操作: `result = ((default(T) == null) ? Activator.CreateInstance<T>() : default(T));` 异常时同样此类操作.
- 数组转换成对应字符分割的字符串:
    - 采用方式`public static string ArrayToString<T>(this IEnumerable<T> array, string sp = ",")`. 循环连接, 去除最后一位.
- Store, Model, Form 三种JSON内容的封装.
    - 未明白的部分: `jsonResult.set_JsonRequestBehavior(0);` 完成方法Allow Get的方式?
    - `public static JsonResult Model(this Controller controller, object obj)` 方式最简单, 将obj JSON序列化后 返回 jsonResult 结果.
    - `public static JsonResult Store(this Controller controller, object _data, object total = null)` 方式返回数量和对象序列化内容. `jsonResult.set_Data()` 设置 data和total的数据节点.
    - `public static JsonResult Form(this Controller controller, bool success = true, object jsonObj = null, string msg = "")` 方式返回success,msg及对象序列化内容.
